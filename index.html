About Us

We aim to successfully develop a Beit Almonah (BA) over the course of 6 weeks, utilizing the MERN stack, which includes Node.js, React, MongoDB, and Express technologies.
This project will be a collaborative effort involving the team members Mohamad Rammal, Riham Khafaja, Zainab Kraki, and Salma Slim.

Week 1

Project Planning

We used Figma and Miro to create UML diagrams for our Beit Almonah project. Figma allowed us to collaborate in real-time, making it easy for our team to work together on class diagrams, sequence diagrams, and use case diagrams. This collaborative interface fostered teamwork and ensured that everyone was on the same page when it came to visualizing our project's architecture.

Plan of design

Miro offered a digital whiteboard experience, perfect for visualizing complex systems and workflows. With its infinite canvas and extensive library of shapes, we could map out our project's structure effectively. By using these tools, we streamlined our design process, fostered collaboration, and ensured everyone understood our project's architecture.

Week 2

Back-End

For the backend of our Beit Almonah project, we used Node.js and Express.js to build a robust API. We defined endpoints for CRUD operations on our MongoDB database and implemented authentication to secure sensitive data. This backend served as the foundation of our project, handling data storage, retrieval, and processing.

Project Pages Distribution Part 1

By structuring our backend with modular components and following MVC architecture, we ensured scalability and maintainability. This modular approach made our codebase easier to understand and maintain, allowing us to iterate and add new features with ease.

Week 3

Front-End

In the frontend of our Beit Almonah project, we created a responsive UI with React.js. We built reusable components and used state management techniques to provide dynamic user experiences. React's component-based architecture allowed us to break down complex UIs into manageable parts, making development more efficient.

Project Pages Distribution Part 2

With modern CSS frameworks like Tailwind, we styled our components for consistency across devices. These frameworks provided pre-built components and utilities that helped us design a visually appealing and user-friendly interface.

Continue CSS Design

Maintain a style guide for cohesive design, and test across devices for a seamless user experience.

Week 4

Front-End Continue

Continuing with our frontend development, we integrated third-party APIs for geolocation and social media authentication. We also enhanced navigation with React Router and implemented client-side form validation for data integrity. These additional features enhanced the user experience and made our application more robust.

Check And Review

By iterating on user feedback, we delivered a polished application that met our users' needs. This iterative approach allowed us to continuously improve our frontend interface and address any issues or concerns raised by our users.

Finish Front-End

Test thoroughly for usability and polish until ready for deployment.

Week 5

Manage State

To manage state in our Beit Almonah project, we used Context API and Redux. Context API helped us share state between components without prop drilling, while Redux offered a centralized store for complex state changes. This combination of state management solutions allowed us to handle state in a more organized and efficient manner.

Mange State Continue

By incorporating these solutions, we simplified state management and improved code maintainability. This made it easier for our team to work on different parts of the application independently, without worrying about how changes in one part might affect another.

Week 6

Web Presentation

For our Beit Almonah project presentation, we showcased our work to stakeholders and potential users. We used visual aids like slide decks and live demos to highlight key features and demonstrate our project's value. This visual presentation helped our audience understand the capabilities of our MERN stack project and how it could benefit them.

Record Videos

Project videos serve as comprehensive visual guides, presenting the Beit Almonah's features, user workflows, and system functionalities, offering an engaging and informative resource for stakeholders and potential users.

Training

Training sessions focus on honing both individual and team presentation abilities, equipping us to articulate the intricacies of the Beit Almonah project confidently to diverse audiences.

Finish Project

Congratulations on completing the Beit Almonah project!

Our Features

ClickUp

ClickUp streamlines our task management with its intuitive interface, fostering efficient organization and timely task completion. Customizable workflows adapt to our library's dynamic needs, enhancing project coordination.

Figma

Figma enables seamless collaboration, aiding our team in real-time design work. Its prototyping and wireframing tools refine our interface, while version control keeps our project history intact and allows for easy reversions.

GitHub

GitHub's secure version control hosts our code repositories, facilitating smooth collaboration among developers. Its issue tracking ensures prompt bug resolution, while pull requests and reviews maintain code quality.

Miro

Miro's visual workspace aids in brainstorming and planning our library system's features collaboratively. Flow diagrams and mind maps create a clear blueprint for development, while remote collaboration fosters team contribution from anywhere.

VS Code

VS Code offers a robust environment for code writing and debugging, enhanced by versatile extensions. Its integrated terminal streamlines workflow, allowing seamless command execution within the editor.

FontAwesome

FontAwesome's diverse icon collection elevates our interface, ensuring visual appeal and usability. Scalability guarantees consistent high-quality visuals across screens, easily integrated within HTML, CSS, and JavaScript.
